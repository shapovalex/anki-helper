# Help Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dedicated Help page explaining the required Anki note type name and field names.

**Architecture:** A new static HTML page (`static/help.html`) served by a new FastAPI route at `/help`, with a "Help" nav link added to the shared menu component. No backend logic — pure static content.

**Tech Stack:** HTML, FastAPI (FileResponse), existing CSS theme (`/static/css/theme.css`)

---

### Task 2: Create the Help page

**Files:**
- Create: `static/help.html`

- [ ] **Step 1: Create the file**

Create `static/help.html` with this content:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Help — Anki Helper</title>
    <link rel="stylesheet" href="/static/css/theme.css">
</head>
<body>
    <div id="menu-container"></div>
    <main class="page page-sm">
        <div class="page-header">
            <h1 class="page-title">Help</h1>
            <p class="page-subtitle">How to set up Anki for use with this app.</p>
        </div>

        <p class="section-heading">Note Type Setup</p>

        <p>Create a note type in Anki whose name exactly matches the <strong>Note Type Name</strong> configured in <a href="/settings">Settings</a> (default: <code>French-Russian</code>). The note type must have exactly these five fields, in any order:</p>

        <ul>
            <li><code>french_word</code></li>
            <li><code>russian_word</code></li>
            <li><code>example</code></li>
            <li><code>french_word_audio</code></li>
            <li><code>example_audio</code></li>
        </ul>
    </main>

    <script src="/static/js/menu.js"></script>
</body>
</html>
```

- [ ] **Step 2: Commit**

```bash
git add static/help.html
git commit -m "feat: add Help page with note type setup instructions"
```

### Task 3: Add /help route in FastAPI

**Files:**
- Modify: `app/main.py`

- [ ] **Step 1: Add the route**

Open `app/main.py`. After the existing `/settings` route at the bottom, add:

```python
@app.get("/help")
async def help_page() -> FileResponse:
    return FileResponse("static/help.html")
```

- [ ] **Step 2: Verify the server starts without errors**

Run:
```bash
uv run uvicorn app.main:app --reload
```

Expected: server starts, no import or syntax errors printed.

- [ ] **Step 3: Verify the page loads**

Open `http://localhost:8000/help` in a browser.

Expected:
- Page renders with the nav menu, "Help" link highlighted as active
- "Note Type Setup" section is visible with the five field names listed

- [ ] **Step 4: Commit**

```bash
git add app/main.py
git commit -m "feat: add /help route"
```
